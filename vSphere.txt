・ ESXi
ハイパーバイザ型ソフトウェア
ESXi をそれぞれの物理サーバーにインストールしておくことで、
１つの物理サーバー上で複数の OS を動かすことが可能。

ハイパーバイザとは？
仮想化ソフトウェアのこと
ホスト OS の代わりにハードウェア上で直接動作する仮想化のための OS 


・ vCenter Server
vSphere 環境の管理一元化を行う

役割
・統合管理（複数のESXi を束ねて管理）
・vSphereにある様々な機能を有効化( vMotion や HA 、 DRS …等々、 vCenter がないと使用できません)

使用方法は2通り
①1台の物理サーバーに vCenter Server をインストールして使う
②仮想マシンに vCenter server をインストールして使う
※vCenterのアプライアンスサーバも用意


・ vSphere Client / Web Client
vSphere への入り口となるインタフェースを提供

vSphere Client
Windows マシンにインストールして使用

vSphere Web Client 
ブラウザベースの vSphere 環境の管理ツールを提供
今後は Web Client に統一される


・ 仮想マシン

物理サーバ＝物理ホスト


仮想マシン　　OS・アプリケーションを入れる器　
ESXiサーバ　　物理サーバ←ESXiインストール

仮想マシンの実態は「ファイル」
仮想 CPU 、仮想メモリ、仮想 HDDが中に存在

仮想マシンの情報はファイルとして「ストレージ」に保存されている

★メリット
実態がファイルなので、仮想マシンの複製が簡単に行える
⇒遠隔地に同じ構成で作成することも、バックアップを取ることも容易になる
	⇒構築時間のコスト削減に繋がる
 　

・ 共有ストレージ
仮想ディスクファイルが保存され、それを読み込むことで仮想マシンを動かしている。

接続方法： FC 、 iSCSI 、 NFS など選択可能


★ネットワーク

・NIC
ネットワークの接続を行うため、 
NIC も他のハードウェアと同様に
仮想的なハードウェアとして仮想マシンに搭載する事が可能

・vNIC（仮想NIC）
ゲスト OS は本物の NIC だと思い、 
このvNIC に IP アドレスを割り当て、通信を行う

通信時にESXiサーバに搭載された物理NICとの紐付けが必要になる。
一つずつ割り当てとなると膨大なNICが必要になる為、
ESXi サーバ内部で仮想スイッチ（vSwitch）を作り、ネットワークコントロールをしている。
